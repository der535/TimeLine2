
| Параметр          | Описание                                                                                                                                                                                                                                                                                   |
| ----------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Назначение        | Класс PlayerHealthPresenter - центральный управляющий компонент системы здоровья, связывающий логику и визуальное представление.                                                                                                                                                           |
| Принцип работы    | Реагирует на игровые события (получение урона, перезапуск игры и т. д.), обновляет состояние здоровья согласно правилам игры, синхронизирует данные с визуальным отображением. Контролирует условия, при которых возможны изменения здоровья (например, только в активном игровом режиме). |
| Как использовать? | Разместите на отдельном GameObject и заполните сериализуемые поля. Система автоматически подключит остальные зависимости через Zenject.                                                                                                                                                    |
